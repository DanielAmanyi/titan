# INC-001 — Simulated Node Failure (Baseline)

## Scenario
Planned simulation of worker node failure on titan-worker-01.

## Environment
- Cluster: TITAN (3 nodes)
- Control Plane: titan-cp-01
- Target Node: titan-worker-01

## Procedure
1. Verify cluster state:
   kubectl get nodes

2. Verify workload placement:
   kubectl get pods -o wide

3. Simulate node failure:
   virsh destroy titan-worker-01

4. Observe system behavior:
   kubectl get nodes -w
   kubectl get pods -o wide -w

## Observations (Expected)
- titan-worker-01 transitions from Ready to NotReady
- Pods on the failed node enter Unknown or Terminating state
- Workloads are rescheduled to titan-worker-02
- Control plane remains responsive
